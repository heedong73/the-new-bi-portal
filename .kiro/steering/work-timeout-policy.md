# 작업 진행 방식

Kiro는 모든 작업에서 아래 규칙을 반드시 따른다.

## 2분 이상 걸리는 작업 처리 규칙

작업이 2분 이상 걸리고 있거나, 2분 이상 걸릴 가능성이 높다고 판단되면 계속 자동으로 진행하지 말고 즉시 멈춘다.

그 대신 사용자가 직접 진행할 수 있도록 다음 형식으로 안내한다.

1. 현재 어디까지 진행했는지 요약한다.
2. 막힌 이유 또는 오래 걸리는 이유를 설명한다.
3. 사용자가 PowerShell, 터미널, IDE에서 직접 실행할 수 있는 명령어를 제시한다.
4. 사용자가 확인해야 할 파일 경로와 수정 위치를 알려준다.

## 금지 사항

- 2분 이상 걸리는 작업을 사용자에게 알리지 않고 계속 진행하지 않는다.
- 긴 작업을 여러 단계로 쪼개지 않고 한 번에 끝내려고 하지 않는다.
- 사용자가 직접 할 수 있는 명령어가 있는데도 설명 없이 내부 작업만 계속하지 않는다.

## 기본 응답 방식

복잡한 작업은 먼저 짧은 계획을 말한 뒤 진행한다.
파일을 만들거나 수정할 때는 가능하면 PowerShell 명령어도 함께 제공한다.
